Getting Started

Follow these instructions to run GravityDesk in development mode or package it as a standalone executable.

Prerequisites

  • Node.js (v18.0 or v20.0+)
  • Antigravity CLI (agy.exe) mapped to your system's PATH.

1. Install Dependencies

npm install

2. Run in Development Mode

npm run electron:dev

3. Build Standalone Installer

npm run electron:build

OAuth & Rate-Limit Rotation

GravityDesk implements a resilient, multi-account rotation layer built to prevent agents from stopping mid-workflow due to API quotas.

Google OAuth Flow

When signing in through the settings panel, GravityDesk spins up a local server on port 8888 to capture Google's authentication response. It grabs the offline refresh_token to allow automated, silent token refreshes whenever credential expiration approaches.

Background Account Switching

If an active account hits any of the following boundaries during a terminal session, the application intercepts the error:

  • Rate Limits (429): The CLI runs out of tokens/second or quota.
  • Eligibility Failures: The account requires manual verification or ToS acceptance.

Keyring Integration & CredWriter

The Go-based Antigravity CLI uses a native library (go-keyring) to pull logged-in credentials from Windows Credential Manager.

The Keyring Parsing Issue

Standard Windows commands (like cmdkey) write credential data using UTF-16 character boundaries. This results in corrupted, null-separated strings when parsed by Go's native keyring decoder, preventing the CLI from detecting GUI accounts.

Win32 C# Injector (CredWriter)

GravityDesk solves this by compiling a custom helper binary, CredWriter.exe, which accesses the native Win32 CredWriteW API. It marshals strings to raw UTF-8 byte arrays and matches generic targets sequential structures, satisfying Go's parser perfectly.
